## 为什么 Hello World 之后要学这个

上一章里，我们已经知道：

- 程序可以从入口开始执行
- 程序可以输出一段固定文本

但如果程序只能输出写死的内容，它就还不算真正开始“处理数据”。

下一步最自然的问题就是：

- 程序能不能先记住一个值，再在后面继续使用它？

这就是变量和常量要解决的问题。

## 先看一条最基本的语句

先看下面这行代码：

```swift
var number = 123
```

这一行虽然很短，但它已经包含了本章最核心的语法结构。

你可以先把它拆成三部分来看：

1. `var`
   表示这里要声明一个变量。

2. `number`
   这是变量名，也就是你后面用来引用这个值的名字。

3. `= 123`
   表示把右边的值放进这个变量里。

所以，这整行代码的意思可以先理解成：

- 创建一个名字叫 `number` 的变量
- 把 `123` 放进去

这也是本章最值得先记住的一点：

- 一条像 `var number = 123` 这样的语句，通常同时完成了两件事
- 第一件事是声明
- 第二件事是赋值

## 什么叫“声明”

在这一章里，你会经常看到“声明变量”或“声明常量”这种说法。

这里的“声明”，你可以先简单理解成：

- 正式告诉程序：我要创建一个新的名字，用它来保存数据

例如：

```swift
var number = 123
```

在这行代码里：

- `number` 这个名字被创建出来了
- 从这一刻开始，你后面就可以继续使用 `number` 这个名字

也就是说，“声明”解决的是这个问题：

- 我们之后要用什么名字来表示这个值？

## 什么叫“赋值”

“赋值”指的是：

- 把一个具体的值放进变量或常量里

还是看这行代码：

```swift
var number = 123
```

这里的：

```swift
= 123
```

就是在做赋值。

你可以先把它理解成：

- 把右边的 `123` 放到左边这个名字对应的位置里

所以，对初学者来说，这一行代码可以先分成两层来理解：

- `var number`：在声明
- `= 123`：在赋值

## 声明并赋值，和重新赋值，不是一回事

这两个动作在初学阶段特别容易混在一起。

先看下面的代码：

```swift
var number = 123
number = 456
```

第一行的意思是：

- 声明变量 `number`
- 同时给它一个初始值 `123`

第二行的意思是：

- 不再声明新变量
- 而是把已经存在的 `number` 改成 `456`

所以你可以先记住这个区别：

- 第一次创建名字并放入值：声明并赋值
- 变量已经存在之后再改值：重新赋值

## 什么是变量

变量就是：

- 程序里一个可以保存数据、并且后面还可能被修改的位置

例如：

```swift
var number = 123
```

这表示程序现在有了一个叫 `number` 的变量，里面存着 `123`。

之后如果你想修改它，还可以这样写：

```swift
number = 456
```

这里要注意一个细节：

- 第一行是声明并赋值
- 第二行不是重新声明，而是把新值赋给已经存在的变量

## 什么是常量

常量也是程序里用来保存数据的位置，但它和变量有一个关键区别：

- 常量一旦确定，就不应该再被修改

例如：

```swift
let word = "Hello"
```

这表示：

- 创建一个叫 `word` 的常量
- 把 `"Hello"` 放进去

## let 和 var 的语法区别

在 Swift 中：

- `var` 用来声明变量
- `let` 用来声明常量

看下面这两行：

```swift
var number = 123
let word = "Hello"
```

它们的整体结构其实是一样的：

```swift
关键字 名字 = 值
```

只是关键字不同：

- `var` 表示“这个值以后可能会变”
- `let` 表示“这个值确定后就不改了”

## 重新赋值时，语法是什么

很多初学者会把“声明”和“重新赋值”混在一起。

可以用下面这个例子区分：

```swift
var number = 123
number = 456
```

第一行的意思是：

- 声明变量 `number`
- 初始值是 `123`

第二行的意思是：

- 给已经存在的 `number` 一个新值 `456`

也就是说，重新赋值时，你不需要再写 `var`。

你现在可以先记住一个最基本的使用原则：

- 这个值后面还要改，用 `var`
- 这个值后面不改，用 `let`

## 什么是数据类型

程序里保存的数据不会都长得一样。

例如：

- 名字是文本
- 年龄是整数
- 温度可能带小数
- 开关状态可能是真或假

这些不同种类的数据，就对应不同的数据类型。

在当前阶段，你只需要先认识最基础的几种。

## 最基础的几种数据类型

### String

`String` 表示字符串，也就是文本。

例如：

```swift
let name = "Swift"
```

这里的 `"Swift"` 就是字符串。

### Int

`Int` 表示整数。

例如：

```swift
let year = 2026
```

这里的 `2026` 就是整数。

### Double

`Double` 表示带小数点的数字。

例如：

```swift
let pi = 3.14
```

这里的 `3.14` 就是小数。

### Bool

`Bool` 表示布尔值，也就是真或假。

例如：

```swift
let isPassed = true
```

它通常只有两个值：

- `true`
- `false`

## 变量之间也可以参与运算

到目前为止，我们只是把值保存起来，再用 `print` 把它输出出来。

但程序真正有意义的地方，不只是“记住值”，还在于：

- 可以拿这些值继续计算

这就是变量之间的运算。

## 最基础的数值运算

先看下面这个例子：

```swift
var a = 2
var b = 5
var c = a + b

print(c)
```

这里发生了三件事：

1. `a` 保存了 `2`
2. `b` 保存了 `5`
3. `a + b` 的结果被保存到了 `c`

所以你可以先把这行代码理解成：

```swift
var c = a + b
```

它的意思是：

- 把 `a` 和 `b` 相加
- 再把结果保存到 `c`

如果程序运行正常，你通常会看到：

```text
7
```

## 常见的运算符

在当前阶段，你先认识最基础的四个就够了：

- `+`：加
- `-`：减
- `*`：乘
- `/`：除

例如：

```swift
var a = 2
var b = 5
var c = a + b
var d = a - b
var e = a * b
var f = a / b

print(c)
print(d)
print(e)
print(f)
```

你现在不需要一下子记住所有规则，只需要先知道：

- 变量里保存的数字，也可以像普通数字那样参与运算

## 除和模有什么区别

初学者很容易把“除法”和“取余”混在一起。

你可以先用下面这组例子区分：

```swift
var a = 2
var b = 5

print(a / b)
print(a % b)
```

这里的两个运算虽然都和“除”有关，但含义并不一样。
显然这里的 `a` 和 `b` 都是整数类型。在当前这个整数运算的例子里，除不尽的部分不会直接变成小数，而是会引出“余数”这个概念。

### `/` 表示除法

```swift
let result = 2 / 5
```

这表示：

- `2` 除以 `5` 的整数部分

### `%` 表示取余（一般读作“模”）

```swift
let remainder = 2 % 5
```

这表示：

- `2` 除以 `5` 的余数

所以你可以先这样记：

- `/` 关注的是商
- `%` 关注的是余数

### 一个最直观的理解方式

以 `2 / 5` 为例：

- `2 / 5 = 0 余 2`

所以：

```text
2 / 5 = 0
2 % 5 = 2
```

如果你把它放进程序里：

```swift
var a = 2
var b = 5

print(a / b)
print(a % b)
```

通常会看到：

```text
0
2
```

当前阶段你只需要先建立这个最基本的直觉：

- `/` 用来看除法结果
- `%` 用来看除完之后还剩多少

## 字符串也可以做最基础的拼接

变量之间的运算不只发生在数字上。

例如字符串也可以用 `+` 连接起来：

```swift
var h = "Hello"
var i = " "
var j = "World"
var k = h + i + j

print(k)
```

这里你可以先理解成：

- `+` 在字符串场景下，不是做加法
- 而是把两段文本连接起来

如果你希望中间有空格，也需要自己把空格写进去：

```swift
var h = "Hello"
var i = " "
var j = "World"
var k = h + i + j
```

## 最基础的简写形式

当你开始写变量之间的运算时，还会遇到一种很常见的简写方式。

例如：

```swift
var a = 2
let b = 5

a = a + b
```

这表示：

- 先取出 `a` 原来的值
- 再加上 `b`
- 最后把结果重新放回 `a`

这类写法可以进一步简写成：

```swift
var a = 2
let b = 5

a += b
```

这里你可以先把：

```swift
a += b
```

理解成：

```swift
a = a + b
```

也就是说，它们表达的是同一件事，只是后者写得更短。

同样的思路还可以出现在：

- `a -= b`
- `a *= b`
- `a /= b`

当前阶段你不需要一次记住太多，只要先知道：

- 这是一种更短的写法
- 它本质上还是“先运算，再重新赋值”

## 为什么“运算”会自然引出“类型”

当你开始写：

```swift
var a = 2
var b = 5
var c = a + b
```

或者：

```swift
var h = "Hello"
var i = " "
var j = "World"
var k = h + i + j
```

你会发现一个很重要的事实：

- 程序不是随便拿两个值就能运算的
- 它需要知道这些值到底是什么类型

这是因为：

- `10` 和 `20` 是数字
- `"Hello"` 和 `"Swift"` 是字符串

同样都是 `+`，在不同类型里，它表达的含义也不一样：

- 数字里的 `+` 表示相加
- 字符串里的 `+` 表示拼接

所以，当你开始学习变量之间的运算时，“类型”这个概念就会变得更重要。

## Swift 是怎么知道类型的

看下面这些代码：

```swift
let name = "Swift"
let year = 2026
let pi = 3.14
let isPassed = true
```

虽然你没有显式写出类型，但 Swift 仍然知道：

- `name` 是 `String`
- `year` 是 `Int`
- `pi` 是 `Double`
- `isPassed` 是 `Bool`

这是因为 Swift 会根据右边的值来推断类型。

这个过程叫做类型推断。

当前阶段你只需要先理解到这一步：

- 右边放什么值，Swift 往往就能推断出左边是什么类型

## 如何查看一个变量的类型

如果你想直接看看某个变量当前到底是什么类型，可以使用：

```swift
type(of:)
```

例如：

```swift
let name = "Swift"
let year = 2026
let pi = 3.14
let isPassed = true

print(type(of: name))
print(type(of: year))
print(type(of: pi))
print(type(of: isPassed))
```

这里你可以先这样理解：

- `type(of: name)` 的意思是“查看 `name` 的类型”

如果程序运行正常，你通常会看到类似下面的输出：

```text
String
Int
Double
Bool
```

这可以帮助你更直观地验证前面讲过的类型推断结果。

## 用 print 观察变量和常量

现在我们可以把变量和常量放进 `print` 里：

```swift
var number = 123
let word = "Hello"

print("number的值是：", number)
print("word的值是：", word)
```

如果程序运行正常，你通常会在控制台看到：

```text
number的值是： 123
word的值是： Hello
```

这说明程序已经不只是输出固定文本，而是能输出“保存起来的数据”。

## 一个最小示例

下面这段代码可以帮助你把本章内容连起来理解：

```swift
var number = 123
let word = "Hello"

print("变量的调用：")
print("number的值是：", number)
print("word的值是：", word)
```

你现在需要重点看懂的是：

- 变量和常量可以先保存值
- 后面可以通过名字把它们取出来
- `print` 不只能输出固定字符串，也能输出已经保存的数据
