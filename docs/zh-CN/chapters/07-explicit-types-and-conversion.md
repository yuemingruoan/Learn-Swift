## 从上一章的习题开始

上一章我们已经知道：

- Swift 可以根据右边的值推断最基础的类型
- `Int`、`Double`、`String`、`Bool` 是最常见的基础类型
- 整数之间可以做加减乘除和取余

但上一章留下了两个很关键的问题：

1. 为什么下面这段代码不能直接通过编译？

```swift
var a
a = 1
```

2. 为什么下面这段代码得不到 `0.4`？

```swift
let a = 2
let b = 5

print(a / b)
```

这两个问题看起来不一样，但它们其实都指向同一个更深一点的概念：

- 编译器需要明确知道一个值到底是什么类型

## 为什么类型推断不是万能的

上一章我们看到过这样的代码：

```swift
let year = 2026
let pi = 3.14
let name = "Swift"
```

这些代码之所以能正常工作，是因为右边的值已经非常明确：

- `2026` 看起来就是整数
- `3.14` 看起来就是小数
- `"Swift"` 看起来就是字符串

所以编译器可以直接推断类型。

但如果你这样写：

```swift
var a
```

情况就不一样了。

这里的问题在于：

- 你只给了编译器一个名字 `a`
- 但没有告诉它 `a` 要保存什么类型的数据
- 也没有给一个初始值让它自动推断

所以在这种情况下，编译器没有足够的信息来判断 `a` 的类型。

## 什么是显式类型声明

当编译器不能自动推断，或者你希望把类型写得更清楚时，就可以使用显式类型声明。

最基础的写法是：

```swift
var a: Int
```

你可以先把这行代码拆成三部分来看：

1. `var`
   表示这里要声明一个变量。

2. `a`
   这是变量名。

3. `: Int`
   表示这个变量的类型是 `Int`。

也就是说，这一行代码的意思是：

- 声明一个名字叫 `a` 的变量
- 并明确告诉编译器：它是整数类型

## 最基础的显式类型语法

在当前阶段，你只需要先记住这一条结构：

```swift
关键字 名字: 类型
```

例如：

```swift
var a: Int
let name: String
var pi: Double
let isPassed: Bool
```

你也可以在声明时同时赋值：

```swift
var a: Int = 1
let name: String = "Swift"
```

所以显式类型声明最常见的两种写法是：

```swift
var a: Int
```

和：

```swift
var a: Int = 1
```

再看下面这个例子：

```swift
var exam: Double = 10
print(exam)
```

你可能会看到：

```text
10.0
```

这里很值得注意的一点是：

- 你写进去的值是 `10`
- 但因为变量被明确声明成了 `Double`
- 所以程序会把它按小数类型来处理和输出

这也说明：

- 显式类型声明不只是“写给编译器看”
- 它也会直接影响这个值后续在程序中的表现方式

## 为什么 `var a; a = 1` 不行

现在再回头看这段代码：

```swift
var a
a = 1
```

它的问题不是第二行的 `a = 1`，而是第一行就已经出了问题。

因为在第一行里：

- 变量名有了
- 但类型没有
- 初始值也没有

所以编译器此时没法确定 `a` 应该是：

- `Int`
- `Double`
- `String`
- 还是别的类型

如果你想写“先声明，后赋值”，那最基础的做法就是：

```swift
var a: Int
a = 1
```

这时编译器就能理解：

- `a` 是一个整数变量
- 后面给它赋值 `1` 是合理的

## 显式类型声明并不会取代类型推断

这里要注意一点：

- 显式类型声明不是为了把类型推断完全替换掉

在很多简单场景里，类型推断仍然很好用。

例如：

```swift
let year = 2026
```

这已经足够清楚。

但当你写成“先声明，再赋值”时，显式类型声明就会变得非常重要。

对当前阶段来说，你可以先形成一个简单判断：

- 右边有明确初始值时，类型推断往往够用
- 没有初始值时，往往需要显式写出类型

## 为什么 `2 / 5` 得不到 `0.4`

现在看第二个问题：

```swift
let a = 2
let b = 5

print(a / b)
```

上一章我们已经知道：

- `2` 会被推断为 `Int`
- `5` 也会被推断为 `Int`

所以这里发生的是：

- 两个整数在做除法

在当前这个语境里，整数除法只会保留整数结果部分，所以：

```text
2 / 5 = 0
```

而不是：

```text
0.4
```

这说明一个很重要的事实：

- 运算结果不只和数值有关
- 还和参与运算的类型有关

## 什么是类型转换

如果你想让这次运算以“小数”的方式进行，就要把整数转换成小数类型。

这就是最基础的类型转换。

例如：

```swift
Double(a)
```

你现在可以先把它理解成：

- 把 `a` 这个整数，转换成 `Double` 形式来参与运算

同理：

```swift
Double(b)
```

表示把 `b` 也转换成 `Double`。

## 最基础的整数转小数

看下面这个例子：

```swift
let a = 2
let b = 5

print(Double(a) / Double(b))
```

这时你可以这样理解：

1. `a` 原本是 `Int`
2. `b` 原本也是 `Int`
3. `Double(a)` 把 `a` 转成了 `Double`
4. `Double(b)` 把 `b` 也转成了 `Double`
5. 现在参与运算的是两个 `Double`

所以程序就会按小数运算的方式处理它们。

通常你会看到：

```text
0.4
```

## Double 和 Int 之间也可以互相转换

类型转换并不只用于除法。

例如：

```swift
var test: Double = 114.514
var test2: Int = Int(test)
var test3: Double = Double(test2)
```

这里可以分三步来看：

1. `test` 是一个 `Double`
   它保存的是 `114.514`

2. `Int(test)`
   表示把这个 `Double` 转成 `Int`

3. `Double(test2)`
   表示再把整数 `test2` 转回 `Double`

如果你把它们输出出来，通常会看到：

```text
test的值是： 114.514
test2的值是： 114
test3的值是： 114.0
```

这里最值得注意的是：

- `114.514` 转成 `Int` 后，会变成 `114`
- 再转回 `Double` 时，得到的是 `114.0`

也就是说，类型转换并不是“给原值换个标签”这么简单。

在这个例子里，小数部分已经在 `Double -> Int` 的过程中丢失了，所以后面再转回 `Double`，也不会重新变回 `114.514`。

## 对比着看最清楚

把这两种写法放在一起会更直观：

```swift
let a = 2
let b = 5

print(a / b)
print(Double(a) / Double(b))
```

它们通常会输出：

```text
0
0.4
```

这组对比最值得你记住的不是结果本身，而是背后的原因：

- `a / b` 是整数除法
- `Double(a) / Double(b)` 是小数除法

## 只要出现小数，推断结果也会变化

除了显式写出：

```swift
Double(a) / Double(b)
```

你还可以观察下面这些写法：

```swift
var c = 2 / 5
var d = Double(2) / Double(5)
var e = 2 / 5.0
var f = 2.0 / 5
```

这几行代码很适合帮助你观察编译器是如何看待类型的。

通常你可以先这样理解：

- `c = 2 / 5`
  两边都是整数，所以结果仍然按整数除法处理。
- `d = Double(2) / Double(5)`
  两边都显式转换成了 `Double`，所以结果是小数。
- `e = 2 / 5.0`
  右边出现了小数 `5.0`，所以这次运算会按小数方式处理。
- `f = 2.0 / 5`
  左边出现了小数 `2.0`，所以这次运算也会按小数方式处理。

如果你再配合：

```swift
print(type(of: c))
print(type(of: d))
print(type(of: e))
print(type(of: f))
```

通常你会看到：

```text
Int
Double
Double
Double
```

这说明一点：

- 只要参与运算的这一侧已经是小数类型，整个结果通常也会进入小数计算的语境

## 为什么 `Double` 和 `Int` 变量不能直接混算

再看下面这个例子：

```swift
var number1 = 2.0
var number2 = 5
```

这时：

- `number1` 会被推断成 `Double`
- `number2` 会被推断成 `Int`

如果你直接写：

```swift
// var result = number1 / number2
```

编译器会报错。

你现在可以先记住这个现象：

- Swift 不会默认帮你把 `Int` 和 `Double` 自动混在一起运算
- 对变量之间的运算，Swift 默认不会进行隐式类型转换

也就是说，虽然：

```swift
2 / 5.0
```

这样的字面量写法可以成立，但如果你已经把变量分别确定成了不同类型，Swift 通常不会再替你做隐式转换。

这也是这一章很重要的一个结论：

- 字面量参与运算时，编译器有时可以根据上下文完成推断
- 但变量与变量之间一旦类型不同，Swift 通常要求你先把类型处理一致，再去运算

这也是为什么显式类型声明和类型转换会放在同一章里讲：

- 因为类型一旦确定，后续的运算规则也会跟着确定

## 先声明再赋值，也可以和类型结合起来看

如果把本章前半部分和后半部分连起来，你就会看到一个更完整的逻辑：

```swift
var a: Int
a = 1

let x = 2
let y = 5

print(x / y)
print(Double(x) / Double(y))
```

这里同时体现了两件事：

1. 没有初始值时，需要显式写出类型
2. 想得到带小数的结果时，需要让数值以 `Double` 的形式参与运算
