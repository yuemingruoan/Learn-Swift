## 为什么这一章要学命令行 I/O

前面几章里，我们已经学会了：

- 输出固定文本
- 保存值
- 观察类型
- 处理 Optional

但到这里为止，程序里的大部分内容仍然是“提前写死”的。

例如：

- 程序要输出什么，是你提前写好的
- 程序里保存什么值，也是你提前写好的

而一旦开始学习输入与输出，程序就会进入一个新的阶段：

- 你不只是把结果打到控制台
- 还可以从用户那里读入内容

这意味着程序开始具备最基础的“交互”能力。

所以这一章的重点不是新语法有多复杂，而是建立下面这个概念：

- 程序不仅可以自己输出内容
- 还可以接收外部输入，再根据输入做出反应

## 什么是命令行输出

在当前这个控制台项目里，最常见的输出方式就是：

```swift
print("Hello")
```

你可以先把 `print` 理解成：

- 把内容显示到控制台

这也是为什么前面的 `Hello World` 示例能立刻在 Xcode 底部控制台看到结果。

对当前阶段来说，你可以先记住：

- 写在 `main.swift` 里的 `print(...)`
- 运行后通常会显示在控制台区域

## `print` 不只能输出固定文本

前面你已经看到过：

```swift
print("Hello")
```

但 `print` 不只能输出写死的字符串，也可以输出变量。

例如：

```swift
let name = "Swift"
print(name)
```

这里输出的就不再是一段固定文本，而是：

- 变量 `name` 当前保存的值

你也可以把文字和变量一起输出：

```swift
let name = "Swift"
print("name 的值是：", name)
```

这时控制台里通常会看到：

```text
name 的值是： Swift
```

这也是命令行程序最常见的基础输出方式之一。

## `print` 的多个参数

先看下面这段代码：

```swift
let age = 18
print("age 的值是：", age)
```

你可以先简单理解成：

- `print` 可以一次输出多个内容
- 它会按顺序把这些内容显示出来

对初学者来说，当前阶段最重要的不是去记住更细的参数规则，而是先建立这个使用习惯：

- 固定提示文字和变量值可以一起输出

这会比你手动拼很多字符串更直观，也更适合刚入门时观察程序结果。

## 一个很实用的小技巧：输出后先不换行

默认情况下，`print(...)` 输出完内容后会自动换行。

例如：

```swift
print("请输入你的名字：")
```

运行时通常会变成这样：

```text
请输入你的名字：
Alice
```

这没有问题，但有时候你会希望用户直接在提示文字后面输入，例如：

```text
请输入你的名字：Alice
```

这时可以使用 `print` 的 `terminator` 参数：

```swift
print("请输入你的名字：", terminator: "")
```

你现在可以先把它简单理解成：

- 默认的 `print(...)` 会在结尾自动补一个换行
- `terminator: ""` 表示把这个“结尾内容”改成空字符串
- 所以输出后光标会停留在同一行

不过这里还有一个很实用的补充点。

在某些环境里，如果你输出后没有换行，那么这段提示文字可能不会立刻显示在控制台上，而是暂时留在输出缓冲区里。

这时更稳妥的写法通常是：

```swift
print("请输入你的名字：", terminator: "")
fflush(stdout)
```

你现在可以先这样理解：

- `stdout` 表示标准输出，也就是当前控制台输出
- `fflush(stdout)` 表示立刻把这部分输出刷新到屏幕上

所以它常常会和：

```swift
print(..., terminator: "")
```

放在一起使用。

原因很简单：

- 不换行时，提示文字有时不会马上显示
- `fflush(stdout)` 可以让提示更稳妥地先显示出来

## 对比看看：不刷新和刷新有什么不同

先看第一种写法：

```swift
print("请输入你的名字：", terminator: "")

if let name = readLine() {
    print("你好，", name)
}
```

这段代码在很多环境里也能正常工作，但在某些环境里，你可能会遇到下面这种体验：

- 程序已经在等待输入
- 但提示文字还没有及时显示出来

也就是说，控制台可能不是先出现：

```text
请输入你的名字：
```

再等待你输入，而是等你输入完内容后文字才会出现。

再看第二种写法：

```swift
print("请输入你的名字：", terminator: "")
fflush(stdout)

if let name = readLine() {
    print("你好，", name)
}
```

这时更理想的执行顺序通常是：

1. 先显示提示文字
2. 再等待用户输入
3. 用户在同一行里输入内容

如果用户输入：

```text
Alice
```

那么通常会看到：

```text
请输入你的名字：Alice
你好， Alice
```

所以这里真正想解决的问题不是“程序能不能读取输入”，而是：

- 提示文字能不能更及时地先显示出来

要特别说明一点：

- 不同环境的缓冲行为可能不同
- 在某些终端里，你可能会觉得“加不加 `fflush(stdout)` 看起来都一样”
- 这也是正常的

但从命令行交互习惯来说，如果你用了：

```swift
print(..., terminator: "")
```

并且希望提示先稳定地显示出来，那么补上一句：

```swift
fflush(stdout)
```

通常会更稳妥。

需要先说明一点：

- 对当前阶段来说，你只需要先记住 `terminator: ""` 这个最常见用法
- 以及在需要时补上一句 `fflush(stdout)`
- 不需要现在就深入研究 `print` 的全部参数和输出缓冲的更深规则

## 什么是命令行输入

如果说 `print(...)` 是把内容送到控制台，那么命令行输入就是反过来：

- 从控制台读取用户输入的内容

在 Swift 的控制台程序里，最常见的基础写法是：

```swift
let input = readLine()
```

你可以先把 `readLine()` 理解成：

- 从当前输入位置读取一整行文本

也就是说，用户在控制台里输入一些内容，再按一次回车后，程序就会尝试把这一整行读进来。

## 为什么 `readLine()` 的结果是 `String?`

这里正好会用到上一章的 Optional。

```swift
let input = readLine()
```

这一行里的 `input` 不是普通的 `String`，而是：

```swift
String?
```

原因是：

- 编译器不能保证每次读取输入时，都一定能拿到一个有效字符串

所以 `readLine()` 返回的不是“必定存在的字符串”，而是：

- 可能读到了一行文本
- 也可能没有读到值

这也是为什么上一章 Optional 要放在这一章前面。

## 最基础的输入示例

先看下面这段代码：

```swift
let input = readLine()

if let text = input {
    print("你输入的是：", text)
}
```

你可以先把它按顺序理解成：

1. 调用 `readLine()` 读取一行输入
2. 得到的结果放进 `input`
3. 因为 `input` 是 `String?`，所以不能直接当普通字符串使用
4. 用 `if let` 先检查里面有没有值
5. 如果有值，就把里面真正的字符串取出来命名为 `text`
6. 再把它输出到控制台

这就是当前阶段最值得先掌握的命令行输入基本模板。

## 一个最简单的交互程序

例如我们想让程序读取一个名字，再输出一句欢迎语：

```swift
print("请输入你的名字：")

if let name = readLine() {
    print("你好，", name)
}
```

如果用户输入：

```text
Alice
```

那么控制台里通常会看到：

```text
请输入你的名字：
你好， Alice
```

这段代码虽然很短，但它已经体现了命令行 I/O 的完整流程：

- 先提示用户输入
- 再读取输入
- 最后根据输入输出结果

## 用不换行提示来做交互

如果你希望输入体验更接近常见命令行程序，也可以把提示和输入放在同一行：

```swift
print("请输入你的名字：", terminator: "")
fflush(stdout)

if let name = readLine() {
    print("你好，", name)
}
```

例如如果输入：

```text
Alice
```

那么控制台里通常会看到：

```text
请输入你的名字：Alice
你好， Alice
```

和前面的例子相比，真正变化的只有一点：

- 输出提示时不再自动换行
- 并且用 `fflush(stdout)` 让提示更稳妥地立即显示出来

这虽然只是一个小细节，但对命令行交互体验会有很直接的改善。

## 输入得到的内容，默认都是字符串

这是这一章非常重要的一点。

不管用户输入的是：

- `Hello`
- `123`
- `999`

只要是通过：

```swift
readLine()
```

读进来的，在当前阶段你都应该先把它理解成：

- 一段字符串

也就是说，用户就算输入了：

```text
123
```

程序最先拿到的也不是整数 `Int`，而是：

```swift
"123"
```

这一点如果没建立清楚，后面就很容易混乱。

## 为什么不能把输入直接当数字使用

假设你想读取一个数字：

```swift
if let text = readLine() {
    print(text)
}
```

这里的 `text` 仍然只是字符串。

所以当前阶段你不能先想当然地把它理解成：

- “用户输入了数字，所以这里就是数字类型”

编译器不会自动替你做这件事。

这和前面我们学过的规则是一致的：

- 类型不同的值不能随便混用
- 需要时要显式转换

## 把输入的字符串转换成整数

如果你希望把一段输入文本当成整数使用，最基础的写法是：

```swift
if let text = readLine() {
    let number = Int(text)
    print(number as Any)
}
```

这里的：

```swift
Int(text)
```

结果不是 `Int`，而是：

```swift
Int?
```

因为字符串转整数这件事也可能失败。

例如：

- `"123"` 可以成功转换
- `"Hello"` 就不可以

所以更稳妥的第一步通常是继续配合 `if let`：

```swift
if let text = readLine() {
    if let number = Int(text) {
        print("转换成功，值是：", number)
    }
}
```

你现在可以先把这段代码理解成：

- 先保证读到了输入文本
- 再尝试把这段文本转换成整数
- 只有转换成功后，才继续使用这个整数

## 一个完整一点的小例子

下面这个例子会读取一个数字，并输出它的两倍：

```swift
print("请输入一个整数：")

if let text = readLine() {
    if let number = Int(text) {
        print("你输入的数字是：", number)
        print("它的两倍是：", number * 2)
    }
}
```

如果用户输入：

```text
21
```

那么通常会看到：

```text
请输入一个整数：
你输入的数字是： 21
它的两倍是： 42
```

这段代码里最关键的逻辑有两层：

1. 先确认 `readLine()` 真的读到了内容
2. 再确认这段内容真的能转换成整数

这也是命令行输入为什么会和 Optional 紧密连在一起的原因。

## 常见问题

### 1. 忘记 `readLine()` 的结果是 Optional

很多初学者会直接把它当成普通字符串使用。

但你要先记住：

- `readLine()` 的结果是 `String?`
- 真正使用前，通常要先处理 Optional

### 2. 以为用户输入数字，程序拿到的就是数字

例如用户输入：

```text
123
```

程序最先拿到的仍然是字符串 `"123"`，不是整数 `123`。

### 3. 忘记 `Int(text)` 也会得到 Optional

很多人会觉得：

- “都写了 `Int(...)` 了，那结果应该已经是整数了”

但实际上：

- `Int(text)` 的结果是 `Int?`
- 因为转换有可能失败

### 4. 输入无效内容后，没有任何输出

例如：

```swift
if let text = readLine() {
    if let number = Int(text) {
        print(number)
    }
}
```

如果用户输入的是：

```text
Hello
```

那么第二层 `if let` 不会进入，所以控制台里可能什么都没看到。

这不一定说明程序坏了，而是说明：

- 转换失败了
- 所以后面的代码没有执行

### 5. 以为 `print` 一定会自动换行

大多数时候确实如此，但如果你写了：

```swift
print("请输入你的名字：", terminator: "")
```

那么输出后光标会停留在当前行末尾。

这不是异常行为，而是你主动修改了 `print` 的结尾方式。

## 本章小结

学完这一章后，你应该已经理解了：

- `print(...)` 是当前最基础的命令行输出方式
- `readLine()` 是当前最基础的命令行输入方式
- `readLine()` 读取的是一整行内容
- `readLine()` 的结果是 `String?`
- 用户输入的内容默认先按字符串处理
- `Int(...)` 可以尝试把字符串转换成整数，但结果仍然可能是 Optional

这一章真正想建立的是：

- 程序不仅能输出固定结果
- 还可以接收用户输入，并基于输入继续执行后面的逻辑
