## 为什么在命令行 I/O 之后学习这一章

前一章里，我们已经学会了：

- 把内容输出到控制台
- 从用户那里读取一行输入
- 把输入当成字符串处理
- 尝试把输入转换成整数

但如果只会输入和输出，程序仍然很“机械”。

例如：

- 读到什么就原样打印出来
- 成功读取后就往下执行
- 失败时往往只能停在原地

这说明程序虽然已经能“接收输入”，但还不会：

- 根据不同情况走不同分支
- 把同一类操作重复执行多次

而这正是表达式、条件判断和循环要解决的问题。

所以这一章真正要建立的是三件事：

- 表达式会产生结果
- 条件判断决定程序走哪条路径
- 循环让程序重复执行一段逻辑

## 什么是表达式

你可以先把表达式理解成：

- 一段会计算出结果的代码

例如：

```swift
1 + 2
```

这是一条表达式，因为它会得到结果：

```text
3
```

再例如：

```swift
age > 18
```

这也是表达式，因为它同样会得到一个结果，只不过结果不是数字，而是：

- `true` 或 `false`

所以当前阶段你可以先抓住这个判断标准：

- 只要一段代码会算出一个值，它通常就可以先看成表达式

## 最常见的三类表达式

在这一章里，最常见的表达式主要有三类：

### 1. 算术表达式

例如：

```swift
let sum = 2 + 3
let diff = 5 - 2
let product = 4 * 3
let quotient = 8 / 2
let remainder = 9 % 2
```

这些表达式会得到数值结果。

你前面其实已经接触过它们，只是这一章开始，我们会更明确地把它们看成：

- “能算出结果”的代码

### 2. 比较表达式

例如：

```swift
let a = 10
let b = 20

print(a > b)
print(a < b)
print(a == b)
print(a != b)
```

这些表达式的结果不是整数，而是 `Bool`。

也就是说，它们只会得到两种结果之一：

- `true`
- `false`

### 3. 逻辑表达式

例如：

```swift
let age = 20
let hasTicket = true

print(age >= 18 && hasTicket)
print(age < 18 || hasTicket)
print(!hasTicket)
```

这里最常见的三个逻辑运算符是：

- `&&`：并且
- `||`：或者
- `!`：取反

它们常用来把多个条件组合起来。

## `Bool` 是什么，为什么它很重要

前面我们已经接触过 `Bool`：

```swift
let isPassed = true
let isFinished = false
```

在当前阶段，你可以先把 `Bool` 理解成：

- 专门表示“是 / 否”
- 或“真 / 假”的类型

如果你想从“代码是怎么得到 `Bool` 的”这个角度去理解，也可以先看下面这个例子：

```swift
var a = 10 < 20
print(a)
print(type(of: a))
```

这里可以按顺序理解成：

1. `10 < 20` 是一个比较表达式
2. 这个表达式会先算出结果
3. 结果是 `true`
4. 所以变量 `a` 最终保存的是一个 `Bool` 值

这段代码通常会输出：

```text
true
Bool
```

这个例子很适合帮助你建立一个直接印象：

- 比较表达式不只是“做了一次判断”
- 它真的会产生一个 `Bool` 结果

这一章里它特别重要，因为：

- 条件判断最终依赖的就是 `Bool`

例如：

```swift
let score = 75
print(score >= 60)
```

这里的：

```swift
score >= 60
```

最终会得到一个 `Bool` 结果。

而 `if`、`while` 这些结构，就是根据这个结果来决定是否执行代码的。

## 为什么条件里必须是 `Bool`

先看下面这段代码：

```swift
let score = 75

if score >= 60 {
    print("及格")
}
```

这段代码可以成立，因为：

- `score >= 60` 的结果是 `Bool`

也就是说，编译器能够明确知道：

- 条件成立还是不成立

但如果你写成：

```swift
let score = 75

// if score {
//     print("及格")
// }
```

这在 Swift 里就不行。

因为：

- `score` 是 `Int`
- 不是 `Bool`

**Swift 不会像某些语言那样，把非零数字自动当成“真”。**

这一点很值得先记住：

- Swift 的条件判断要求更明确
- 条件表达式必须真正得到 `Bool`

这会让代码更直接，也更不容易产生歧义。

## 最基础的条件判断：`if`

最简单的条件判断是：

```swift
if 条件 {
    条件成立时执行的代码
}
```

例如：

```swift
let score = 75

if score >= 60 {
    print("及格")
}
```

这段代码你可以先这样理解：

- 先计算 `score >= 60`
- 如果结果是 `true`
- 就执行大括号里的代码

也就是说，`if` 最核心的作用是：

- 条件成立才执行

## `if else`

如果你不只是想处理“成立”这一种情况，而是还想处理“不成立”的情况，就可以写成：

```swift
let score = 45

if score >= 60 {
    print("及格")
} else {
    print("不及格")
}
```

你可以先把它理解成：

- 如果条件成立，执行第一块代码
- 否则执行 `else` 里的代码

所以 `if else` 的核心是：

- 两条路只能走一条

## `else if`

如果情况不只两种，而是有多种分支，就可以继续往下写：

```swift
let score = 85

if score >= 90 {
    print("优秀")
} else if score >= 60 {
    print("及格")
} else {
    print("不及格")
}
```

这里的执行顺序要特别注意：

1. 先判断第一条条件
2. 如果不成立，再判断第二条条件
3. 如果前面的都不成立，再进入 `else`

所以 `else if` 可以先理解成：

- 继续加一层新的条件判断

## 条件判断的一个命令行示例

前一章我们已经知道，用户输入的内容通常需要先转换。

现在就可以把输入和条件判断连起来：

```swift
print("请输入一个整数：", terminator: "")
fflush(stdout)

if let text = readLine() {
    if let number = Int(text) {
        if number > 0 {
            print("这是一个正数")
        } else if number < 0 {
            print("这是一个负数")
        } else {
            print("这是 0")
        }
    }
}
```

这段代码的意义在于：

- 先读到输入
- 再尝试转换成整数
- 转换成功后，再根据这个整数进入不同分支

这就是“输入”和“条件判断”开始配合工作的最基础形式。

## 什么是循环

条件判断解决的是：

- 不同情况走不同路径

而循环解决的是：

- 同一类操作要重复做很多次

例如你想：

- 连续输出 1 到 5
- 连续检查用户输入
- 重复提示直到用户输入合法内容

这时就会用到循环。

## `for-in` 循环

对当前阶段来说，最适合先掌握的循环是：

```swift
for 变量 in 范围 {
    重复执行的代码
}
```

例如：

```swift
for i in 1...5 {
    print(i)
}
```

这里你可以先这样理解：

- `i` 会依次取 `1`、`2`、`3`、`4`、`5`
- 每取到一个值，就执行一次大括号里的代码

所以它的输出通常是：

```text
1
2
3
4
5
```

## `...` 和 `..<` 的区别

这一点在 `for-in` 里非常常见。

```swift
for i in 1...5 {
    print(i)
}
```

这里的：

```swift
1...5
```

表示闭区间，也就是：

- 包含 `1`
- 也包含 `5`

而下面这种写法：

```swift
for i in 1..<5 {
    print(i)
}
```

表示：

- 包含左边的 `1`
- 但不包含右边的 `5`

所以它通常会输出：

```text
1
2
3
4
```

当前阶段你只需要先记住：

- `...`：右边也算进去
- `..<`：右边不算进去

## `while` 循环

除了 `for-in`，另一种很常见的循环是：

```swift
while 条件 {
    条件成立时重复执行的代码
}
```

例如：

```swift
var count = 1

while count <= 3 {
    print(count)
    count = count + 1
}
```

这段代码可以先这样理解：

1. 先看 `count <= 3` 是否成立
2. 如果成立，就执行循环体
3. 执行完之后再回头判断一次
4. 直到条件不成立为止

所以 `while` 的核心是：

- 条件成立，就继续重复

## 为什么 `while` 容易写出死循环

还是看上面的代码：

```swift
var count = 1

while count <= 3 {
    print(count)
    count = count + 1
}
```

这里之所以能停下来，是因为：

- `count` 在循环体里不断变化

但如果你忘了更新它，例如：

```swift
var count = 1

while count <= 3 {
    print(count)
}
```

那么条件会一直成立，程序就会反复输出 `1`，很难停下来。

所以对 `while` 来说，一个很重要的检查习惯是：

- 循环里的状态有没有变化

当然XCode也会替你检查并提醒你，但它的检测逻辑并不是十全十美的，不可过度依赖。

## 如何在中途跳出循环：`break`

前面的循环例子里，循环通常都是因为：

- 条件不再成立

才自然结束。

但有时候你会遇到另一种情况：

- 原来的循环条件还成立
- 但你已经找到了想要的结果
- 或者已经满足了提前结束的条件

这时就可以使用：

```swift
break
```

你现在可以先把它理解成：

- 立刻结束当前循环

例如：

```swift
for i in 1...10 {
    if i == 5 {
        break
    }

    print(i)
}
```

这段代码通常会输出：

```text
1
2
3
4
```

因为当 `i == 5` 时，循环会被直接结束，后面的内容就不会再执行。

`break` 在 `while` 里也一样能用。

例如：

```swift
var count = 1

while true {
    print(count)

    if count == 3 {
        break
    }

    count = count + 1
}
```

这里如果没有 `break`，循环会一直继续下去。

而真正让它停下来的，就是：

- 当 `count == 3` 时执行 `break`

所以对当前阶段来说，你可以先记住：

- 循环有两种常见结束方式
- 一种是条件自然变成不成立
- 另一种是在循环内部满足条件后，用 `break` 提前结束

## 如何跳过当前这一轮：`continue`

`break` 的作用是：

- 直接结束整个循环

而 `continue` 不一样。

你现在可以先把它理解成：

- 跳过当前这一轮剩下的代码
- 直接开始下一轮循环

例如：

```swift
for i in 1...5 {
    if i == 3 {
        continue
    }

    print(i)
}
```

这段代码通常会输出：

```text
1
2
4
5
```

因为当 `i == 3` 时：

- 当前这一轮不会继续往下执行 `print(i)`
- 程序会直接进入下一轮循环

所以这里特别适合和 `break` 做一个对比：

- `break`：整个循环直接结束
- `continue`：本轮跳过，下一轮继续

`continue` 在 `while` 里也能使用。

例如：

```swift
var count = 0

while count < 5 {
    count = count + 1

    if count == 3 {
        continue
    }

    print(count)
}
```

它通常会输出：

```text
1
2
4
5
```

这里要特别注意顺序。

在 `while` 里使用 `continue` 时，通常更要小心：

- 会不会把更新状态的代码跳过去

比如上面这个例子里，我先写了：

```swift
count = count + 1
```

再判断是否 `continue`，就是为了保证：

- 即使跳过当前这一轮，`count` 也仍然会继续变化

否则就可能意外进入死循环。

## 条件判断和循环怎么配合使用

如果把这一章和前一章连起来，你就会看到一个更像“真实程序”的结构：

```swift
var isValid = false

while !isValid {
    print("请输入一个整数：", terminator: "")
    fflush(stdout)

    if let text = readLine() {
        if let number = Int(text) {
            print("输入成功，值是：", number)
            isValid = true
        } else {
            print("输入无效，请重新输入")
        }
    }
}
```

这段代码体现了三件事：

1. `while` 负责重复执行
2. `if let` 负责处理输入和转换
3. 条件判断决定这次输入是成功还是失败

这时程序就不再是：

- “执行一次就结束”

而是：

- “直到用户输入合法内容才继续”

这已经非常接近很多真实交互程序的基本结构了。

## 常见错误

### 1. 把 `=` 写成 `==`

例如：

```swift
// if score = 60
```

这里经常会混淆。

你现在只需要先记住：

- `=` 是赋值
- `==` 是判断是否相等

### 2. 条件不是 `Bool`

例如：

```swift
let score = 75

// if score { ... }
```

这在 Swift 里不成立。

因为：

- `score` 是 `Int`
- 不是 `Bool`

### 3. 把 `...` 和 `..<` 搞混

这一点会直接影响循环次数。

例如：

- `1...5` 会执行 5 次
- `1..<5` 会执行 4 次

### 4. `while` 条件一直不变

如果循环体里没有让条件发生变化，就容易进入死循环。

### 5. 以为循环只能等条件失效后才能结束

不一定。

有些时候你已经在循环内部找到了想要的结果，这时可以直接使用：

```swift
break
```

来提前结束循环。

### 6. 使用 `continue` 时，忘了循环状态仍然要变化

这在 `while` 里尤其容易出问题。

如果你在 `continue` 前面没有让循环状态发生变化，那么程序就可能一直卡在同一轮里。

### 7. 以为输入后可以直接比较数字大小

例如：

```swift
if let text = readLine() {
    // 这里的 text 还是 String
}
```

你要先记住：

- 输入默认是字符串
- 真要按数字处理，通常还得先做转换

## 本章小结

学完这一章后，你应该已经理解了：

- 表达式会产生结果
- 比较表达式和逻辑表达式常常会得到 `Bool`
- `if`、`else if`、`else` 用来处理不同分支
- `for-in` 用来按范围重复执行代码
- `while` 用来在条件成立时持续重复
- 条件判断和循环可以和命令行输入组合起来，形成更完整的交互逻辑

这一章真正的关键不是记住多少语法，而是建立下面这条主线：

- 先得到一个结果
- 再根据结果决定走哪条路径
- 必要时把这套逻辑重复执行

## 思考题

请你尝试写一个命令行成绩录入程序，完成下面这些事：

1. 提示用户输入学生姓名
2. 提示用户输入考试次数
3. 考试次数也要先检查是否合法：
   - 必须能够成功转换成整数
   - 并且必须大于 `0`
   - 如果输入非法，就提示重新输入，直到输入合法为止
4. 在考试次数确定后，再依次提示用户输入每一次考试的成绩
5. 每次输入成绩时，都要先检查它是不是合法成绩：
   - 成绩范围必须在 `0...100` 之间
   - 如果不能成功转换成整数，就属于非法输入
   - 如果转换成功了，但分数小于 `0` 或大于 `100`，也属于非法输入
   - 只要出现非法输入，就提示重新输入，直到输入合法为止
6. 当所有考试成绩都输入完成后，输出：
   - 学生姓名
   - 考试次数
   - 总分
   - 平均分
   - 及格率
7. 其中“及格率”要求以百分数形式输出
8. 及格标准按 `60` 分计算，也就是：
   - 分数大于等于 `60`，算及格
   - 分数小于 `60`，算不及格

你可以先把“及格率”理解成：

- 所有考试里，分数大于等于 `60` 的次数
- 除以总考试次数
- 再乘以 `100`
- 最后以百分数形式输出
- 及格率保留一位小数，需要四舍五入

例如，如果考试次数是 `5`，其中有 `4` 次及格，那么：

```text
及格率 = 4 / 5 = 0.8 = 80.0%
```

例如，如果考试次数是 `9`，其中有 `6` 次及格，那么：

```text
及格率 = 6 / 9 = 0.6666666..... = 66.7%
```

你可以参考下面这组样例输入输出，帮助自己理解题目的目标效果。
