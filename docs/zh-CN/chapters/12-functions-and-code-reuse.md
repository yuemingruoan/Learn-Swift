## 什么是函数

在当前阶段，可以先把函数理解成：

- 一段被命名、可以重复调用的代码

这一定义里有两个关键点：

1. 被命名
   也就是说，这段逻辑不再只是散落在代码中的若干行，而是有了一个明确的名字。

2. 可以重复调用
   也就是说，需要这段逻辑时，不必把整段代码重新写一遍，而是直接使用这个名字。

因此，函数的核心价值并不在于“语法看起来更正式”，而在于：

- 同一段逻辑只写一次
- 后面需要时可以多次使用
- 代码更为清晰，便于维护

## 最基础的函数写法

先看一个最简单的例子：

```swift
func sayHello() {
    print("Hello")
}
```

这段代码可以拆成三部分：

1. `func`
   表示这里要定义一个函数。

2. `sayHello`
   这是函数名，也就是这段逻辑的名字。

3. `{ ... }`
   这是函数体，表示这个函数真正要执行的代码。

当前阶段先记住最基础的结构即可：

```swift
func 函数名() {
    函数体
}
```

## 定义函数，不等于执行函数

这是初学阶段非常容易混淆的一点。

例如：

```swift
func sayHello() {
    print("Hello")
}
```

这段代码只是“定义”了一个函数，并不会自动执行。

如果要真正运行它，还需要再写：

```swift
sayHello()
```

完整写法如下：

```swift
func sayHello() {
    print("Hello")
}

sayHello()
```

此时程序才会真正输出：

```text
Hello
```

因此，这里需要先建立一个明确区分：

- 定义函数：告诉程序“这段逻辑以后可以这样调用”
- 调用函数：真正执行这段逻辑

## 带参数的函数

前面的 `sayHello()` 有一个明显限制：

- 它只能输出写死的内容

如果希望函数处理不同输入，就需要参数。

例如：

```swift
func greet(name: String) {
    print("你好，\(name)")
}
```

这里的：

```swift
name: String
```

就是函数的参数。

它表示：

- 这个函数在执行时，需要先得到一个 `String`
- 这个字符串在函数体内部被命名为 `name`

调用时可以写成：

```swift
greet(name: "Alice") //
greet(name: "Bob") //
```

这样同一个函数就可以处理不同输入。

因此，参数的核心作用是：

- 让函数不再依赖写死的固定值
- 让同一段逻辑可以在不同输入下重复使用

## 参数名和调用标签

看到这里，很多读者会自然形成一个印象：

```swift
func greet(name: String) {
    print("你好，\(name)")
}
```

是不是就等于：

- `name` 是参数名
- 调用时也直接写 `name:`

在当前阶段，这样理解当然没问题。但当你后面读到系统 API 时，很快就会看到更完整的写法：

```swift
func 函数名(外部名字 内部名字: 类型) {
    // ...
}
```

这时要先建立一个新概念：

- **外部参数名**
  - 调用函数时写在前面的标签

- **内部参数名**
  - 函数体内部真正使用的变量名

例如：

```swift
func greet(to name: String) {
    print("你好，\(name)")
}
```

这里可以拆成：

- `to`
  - 外部参数名
  - 调用时要写出来

- `name`
  - 内部参数名
  - 函数体里真正使用它

所以调用时会写成：

```swift
greet(to: "Alice")
```

而函数体内部仍然写：

```swift
print(name)
```

### 为什么要分成两个名字

因为有时“调用时的可读性”和“函数体里的命名”并不完全是一回事。

像：

```swift
greet(to: "Alice")
```

读起来更像一句自然语言。

但在函数体内部：

```swift
name
```

又比 `to` 更适合拿来表示“这个字符串本身是什么”。

### 前面为什么没单独讲这个

因为在最基础的入门写法里，Swift 经常会让外部名字和内部名字看起来像是同一个东西，例如：

```swift
func greet(name: String) {
    print(name)
}
```

你当前阶段可以先把它近似理解成：

- 调用时用 `name:`
- 函数体里也用 `name`

但后面一旦碰到系统 API，你就要开始有一个意识：

- 参数声明里，冒号前面不一定只有一个名字

这个知识点现在先记住就够了。后面读到像：

```swift
func data(from url: URL) async throws -> (Data, URLResponse)
```

这种系统函数时，就不会疑惑：

- 为什么 `from` 和 `url` 好像都出现在参数位置

因为它表达的正是：

- `from`
  - 调用时使用的外部参数名

- `url`
  - 函数内部使用的参数变量名

## 用 `_` 省略外部参数名

前面讲的是：

- 有些参数会有外部参数名
- 调用时要把这个标签写出来

但 Swift 里还有另一种情况：

- **调用时故意不写外部参数名**

这时就会用到 `_`。

例如：

```swift
func add(_ number1: Int, _ number2: Int) -> Int {
    return number1 + number2
}
```

这里的两个 `_` 表示：

- 第一个参数调用时不写标签
- 第二个参数调用时也不写标签

所以调用时会写成：

```swift
add(1, 2)
```

而不是：

```swift
add(number1: 1, number2: 2)
```

这时要先建立一个很重要的理解：

- `_` 不是“没有参数”
- `_` 也不是“参数名叫下划线”
- 它表达的是：**这个参数没有外部参数名**

函数体内部照样会有内部参数名可以使用：

- `number1`
- `number2`

所以你可以把它理解成：

- 调用时省略标签
- 函数体里照常有变量名可用

后面当你读到系统 API 时，经常会看到这种混合写法，例如：

```swift
setValue(_:forHTTPHeaderField:)
```

这种写法通常是文档里对“方法外形”的简写，它强调的是：

- 第一个参数调用时不写标签
- 第二个参数调用时使用 `forHTTPHeaderField:`

如果把它近似展开成完整声明，可以先理解成：

```swift
func setValue(_ value: String?, forHTTPHeaderField field: String)
```

这就表示：

- 第一个参数没有外部参数名
- 第二个参数的外部参数名是 `forHTTPHeaderField`

所以调用时才会写成：

```swift
setValue("application/json", forHTTPHeaderField: "Content-Type")
```

当前阶段先记住这一条规则就够了：

- 看到 `_`，就想到“这个参数调用时不写标签”

## 有返回值的函数

观察前面举例的函数，不难发现，它只是自顾自地执行了里面的代码，但和外界没有任何交流

在先前我们已经学过如何使用`readLine()`函数：

```swift
if let text = readLine() {
    print("你输入的是：\(text)")
}
```

让我们暂时忽略`Optional`,来看这部分：

```swift
text = readLine()
```

显然，这是一条最基本的赋值语句，它的含义也很简单：

- 从控制台中读取一行输入，将其赋值给`text`变量

但是让我们深挖一下这个等式：

- 等号左边：被赋值的对象，也就是`text`变量，很合理

但当我们看到等号右边时，问题便出现了：

- `text`被赋予的值是从控制台读取的那一行输入
- 但等式右边却是`readLine()`函数

显然这里发生了什么，让`readLine()`函数变成了读取到的输入内容 

也就是说这里实际上还有一行隐藏的”代码“:

```swift
readLine() = WHAT_YOU_INPUT(你输入的内容)
```

如果你不理解其中的含义，我们可以举一个数学中的例子

设`f(x) = 2x`

易得：
- f(1) = 2
- f(2) = 4
- f(3) = 6
- ......

显然，当你计算`f(3)+5`的值时，你会很自然地开始计算`6+5=11`

这很合理，因为6就是f(3)的值，所以你会很自然地把6代入算式，就像......

**text就是readLine()的值！**

而在编程中，我们称它为`返回值`

但是这就又引出了一个问题：函数该怎么告诉编译器，我要返回什么呢？

来看下面这段代码：

```swift
func add(number1:Int,number2:Int)->Int{
    var result:Int = number1+number2
    return result
}
```

显然这是一个极其简单的函数

- 首先，它接受了`number1`和`number2`两个`Int`类型的参数 
- 然后，它完成了相加操作，并将结果存到了一个新的变量`result`中

重点在于最后一行，`return result`,它的含义是

- 返回`result`变量**的值**
- 或者说，将`result`变量作为函数的返回值

例如，当你执行以下代码时：
```swift
print(add(number1: 1, number2: 2))
```

程序实际上做了这些操作：

- 将参数`1`和`2`分别赋值给`number1`和`number2`两个变量
- 执行`add`函数体内的代码，即`var result:Int = number1+number2`
- (显然，此时`result`变量的值为`3`)
- 返回`result`变量的值
- 外层的`print()`函数收到了结果`3`,作为自身的参数
- (此时的语句等价于`print(3)`)
- `print()`函数执行内部语句，将`3`输出到控制台中

直到此处，我们终于可以揭晓定义函数的通用语法了：

```swift
func 函数名(参数一:类型,参数二:类型,参数三:类型, ......) -> 返回值类型（若有返回值）{
    //你需要执行的代码
    return 返回值（若有）
}
```

## 一个函数返回多个值：元组

前面看到的函数都只返回一个结果，例如：

- 一个 `Int`
- 一个 `Bool`
- 一行输入文本

但有些场景里，函数结束后你希望同时拿回两项结果。

例如：

- 总分和平均分
- 商和余数
- 姓名和年龄

这时就可以先用元组。

当前阶段可以把元组理解成：

- 把几个值临时打包在一起

最简单的写法如下：

```swift
let student = ("Alice", 20)
```

这里的意思是：

- 这个值里一共装了两项内容
- 第一项是 `"Alice"`
- 第二项是 `20`

### 元组的类型长什么样

如果把类型也写出来，它会长这样：

```swift
let student: (String, Int) = ("Alice", 20)
```

这表示：

- 这是一个元组
- 第一项类型是 `String`
- 第二项类型是 `Int`

也就是说，元组的类型本身也会按顺序写出每一项的类型。

### 怎样读取元组里的值

最基础的方式是按位置取值：

```swift
let student = ("Alice", 20)

print(student.0)
print(student.1)
```

这里的：

- `.0` 表示第一个值
- `.1` 表示第二个值

输出通常是：

```text
Alice
20
```

### 怎样把元组拆开

更常见、也更适合后面章节的写法，是直接拆开：

```swift
let student = ("Alice", 20)
let (name, age) = student

print(name)
print(age)
```

这段代码的意思是：

- 把元组里的第一个值交给 `name`
- 把元组里的第二个值交给 `age`

这种写法通常叫“拆包”或“解构”。

你可以先把它理解成：

- 一次把元组里的多个值分别取出来

### 函数怎样返回元组

既然元组能把多个值打包在一起，函数就可以把元组当成返回值。

例如：

```swift
func totalAndAverage(scores: [Int]) -> (Int, Double) {
    let total = scores[0] + scores[1] + scores[2]
    let average = Double(total) / 3.0
    return (total, average)
}
```

这里的返回值类型：

```swift
(Int, Double)
```

表示这个函数会同时返回两项结果：

- 一个 `Int`
- 一个 `Double`

调用时可以这样写：

```swift
let result = totalAndAverage(scores: [80, 90, 70])
print(result.0)
print(result.1)
```

也可以直接拆开：

```swift
let (total, average) = totalAndAverage(scores: [80, 90, 70])
print(total)
print(average)
```

这也是当前阶段最值得记住的用法：

- **当函数需要一次交出多个结果时，可以先用元组**

### 元组和后面要学的 `struct` 有什么区别

最简单的判断方式就是：

- 元组适合临时把几个值放在一起
- `struct` 更适合表达有明确含义、会反复使用的一组数据

例如：

- “函数需要一次性返回总分和平均分”
  用元组就够了

- “学生记录、任务信息、课程数据”
  这类更稳定的数据模型，后面更适合用 `struct`

## 从第十章思考题回看重复代码

在第十章思考题的参考答案中，最值得注意的不是某一条具体语法，而是重复逻辑出现的位置。

例如，可以很容易看到下面几类重复：

### 1. 输入提示的重复

程序中多次出现了类似这样的代码：

```swift
print("请输入学生姓名：", terminator: "")
fflush(stdout)
```

或者：

```swift
print("请输入考试次数：", terminator: "")
fflush(stdout)
```

它们的区别只在提示文字本身，而输出方式其实是同一类逻辑。

### 2. 非法输入提示的重复

程序中多次出现：

```swift
print(invalid_prompt)
```

这种逻辑虽然很短，但本质上也是重复。

### 3. 判定逻辑的重复

例如：

- 判断考试次数是否合法
- 判断成绩是否在 `0...100` 范围内
- 判断是否及格

这些都属于“同一类判断逻辑”。

### 4. 小数处理逻辑的重复

平均分和及格率都要保留一位小数，这说明：

- 同一套处理规则被使用了不止一次

只要出现这种现象，就说明这段逻辑已经开始具备“函数化”的价值。

## 提炼第一个函数：统一输出输入提示

先从最简单的一类重复开始。

例如可以先写：

```swift
func printPrompt(message: String) {
    print(message, terminator: "")
    fflush(stdout)
}
```

这个函数的意义很直接：

- 所有“输出提示但不换行”的代码，都可以交给它处理

这样原来的代码：

```swift
print("请输入学生姓名：", terminator: "")
fflush(stdout)
```

就可以改成：

```swift
printPrompt(message: "请输入学生姓名：")
```

而：

```swift
print("请输入考试次数：", terminator: "")
fflush(stdout)
```

就可以改成：

```swift
printPrompt(message: "请输入考试次数：")
```

这一类提炼的优点是：

- 语义更集中
- 后面如果提示输出方式要调整，只改一处即可

## 提炼第二个函数：判断成绩是否合法

再看这一类逻辑：

```swift
if score_Int_temp <= 100 && score_Int_temp >= 0 {
    ...
}
```

这一段判断也可以独立提炼。

例如：

```swift
func isValidScore(score: Int) -> Bool {
    return score >= 0 && score <= 100
}
```

这样之后就可以写成：

```swift
if isValidScore(score: score_Int_temp) {
    ...
}
```

这个函数的价值在于：

- 把“合法成绩”的定义单独命名

这样读代码时，不需要每次重新解析 `>= 0 && <= 100` 的条件组合，只需要看到：

- 这里在判断成绩是否合法

## 提炼第三个函数：判断是否及格

这也是一个非常适合提炼的逻辑。

例如：

```swift
func isPassed(score: Int) -> Bool {
    return score >= 60
}
```

这样原来的代码：

```swift
if score >= 60 {
    pass_num += 1
}
```

可以改成：

```swift
if isPassed(score: score) {
    pass_num += 1
}
```

这一改动的意义并不在于“少写了几个字符”，而在于：

- `60` 分这条业务规则被明确命名了

这会让代码更接近需求本身。

## 提炼第四个函数：保留一位小数

前一章中已经说明，平均分和及格率都需要保留一位小数。

既然同一套规则被使用了两次，就应当考虑函数化。

例如可以写成：

```swift
func roundToOneDecimal(value: Double) -> Double {
    var firstDecimal = Int(value * 10)
    let secondDecimal = Int(value * 100) % 10

    if secondDecimal >= 5 {
        firstDecimal += 1
    }

    return Double(firstDecimal) / 10
}
```

这个函数内部做的事情并不复杂：

1. 先把第一位小数移动到整数部分
2. 再取出第二位小数
3. 根据第二位小数决定是否进位
4. 最后再把小数点还原

这样后续就可以直接写成：

```swift
average = roundToOneDecimal(value: Double(score_sum) / Double(frequency))
pass_rate = roundToOneDecimal(value: Double(pass_num) / Double(frequency) * 100)
```

与前一章相比，这里的关键变化是：

- 保留一位小数的规则不再分散在两个位置
- 这套逻辑被统一收进了一个函数

## 为什么函数会让主流程更清楚

当重复逻辑被函数提炼出去之后，主流程会明显更接近需求本身。

例如，下面这种写法：

```swift
printPrompt(message: "请输入考试次数：")

if let frequency_String_temp = readLine() {
    if let frequency_Int_temp = Int(frequency_String_temp) {
        if frequency_Int_temp > 0 {
            frequency = frequency_Int_temp
            break
        } else {
            print(invalid_prompt)
        }
    } else {
        print(invalid_prompt)
    }
}
```

虽然仍然有多层判断，但相比完全展开的写法，已经更容易看出主线：

- 先输出提示
- 再读取输入
- 再做转换与校验

如果继续提炼，主流程还可以进一步简化。

这正是函数的真正价值：

- 不只是减少重复
- 还会让主流程结构更加清楚

## 本章最需要建立的三个判断标准

学到这里，最重要的不是记住多少函数例子，而是建立下面三个判断标准。

### 1. 这段逻辑是否重复出现

如果一段逻辑已经出现两次及以上，就值得考虑提炼成函数。

### 2. 这段逻辑是否有明确职责

如果一段逻辑能够用一句短话说明，例如：

- 输出输入提示
- 判断成绩是否合法
- 判断是否及格
- 保留一位小数

那么它通常已经具备函数化的条件。

### 3. 提炼后，主流程是否更清晰

函数并不是越多越好。

是否值得提炼，最终仍然要看：

- 提炼之后，主流程是不是更接近需求本身

如果提炼后只是把简单代码打碎，反而更难读，那么就没有必要。

## 常见错误

### 1. 只定义函数，不调用函数

定义函数之后，如果没有调用，它不会自动执行。

### 2. 参数类型和传入值不匹配

例如：

```swift
isPassed(score: "80")
```

这里就会出错，因为函数期望的是 `Int`，不是 `String`。

### 3. 该返回结果时，只写了 `print(...)`

例如：

```swift
func add(a: Int, b: Int) -> Int {
    print(a + b)
}
```

这类写法混淆了“输出结果”和“返回结果”。

### 4. 函数职责不清

如果一个函数既负责输入、又负责计算、又负责输出，当前阶段往往会让结构更混乱。

初学阶段更适合先提炼职责单一的小函数。

## 本章小结

本章介绍的重点不是函数语法本身，而是函数在程序组织中的作用。

在当前阶段，最重要的结论可以概括为：

- 函数用于减少重复代码
- 参数用于接收输入
- 返回值用于交出结果
- 提炼函数的目标，是让主流程更清楚，而不是单纯追求“代码更短”

如果能够从第十章思考题中识别重复逻辑，并逐步将其提炼为函数，那么后续进入结构体、类以及更复杂的程序组织方式时，难度会明显降低。
